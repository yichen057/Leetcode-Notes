Q: 本题为啥cur!=pre时, count要归1呢? 为啥只是和相邻元素作比较呢? 

这个问题问到了 **501. 二叉搜索树中的众数**（Find Mode in BST）最核心的技巧：**利用中序遍历的“有序性”来简化计数**。

简单直接的回答是：因为在 **二叉搜索树（BST）** 的中序遍历中，**所有相同的元素一定是被排在一起的**。

------

### 1. 为什么只和“相邻”元素比较？

如果这是一个普通的数组 `[3, 1, 2, 3, 1]`，你确实不能只比相邻的，因为相同的数字可能散落在各处。

但 BST 的中序遍历就像一个**已经排好序的数组**，比如：

```
[1, 1, 2, 3, 3, 3, 4, 4]
```

观察这个有序数组，你会发现：

- 相同的数字（比如那三个 `3`）一定是**紧紧挨着**的。
- 一旦你遇到了一个不一样的数字（比如从 `3` 变成 `4`），那么在这个序列后面的任何地方，**都不可能再出现 `3` 了**。

**结论**：由于“同类必相邻”，我们只需要拿“当前元素”和“它的前一个元素（`pre`）”比。如果一样，计数加 1；如果不一样，说明开启了一个新数字的计数。

------

### 2. 为啥 `cur != pre` 时，`count` 要归 1？

这里的 `count` 记录的是**当前正在统计的这个数字出现的频率**。

还是以 `[1, 1, 2, 2, 2]` 为例：

1. **遇到第一个 1**：`pre` 是 `None`。`count = 1`。
2. **遇到第二个 1**：此时 `cur(1) == pre(1)`。`count` 累加变成 `2`。
3. **遇到第一个 2**：此时 `cur(2) != pre(1)`。
   - **关键点**：这意味着“1”这个数字的统计彻底结束了。
   - **动作**：我们要开始统计“2”出现的次数了。由于“2”现在是第一次露面，它的初始频率当然是 **1**。
   - 这就是为什么要 **`count = 1`**。

------

### 3. 如果不归 1 会发生什么？

如果你不归 1，而是继续累加，那么 `count` 就会变成“到目前为止一共看了多少个数字”，而不是“当前这个数字出现了几次”。这样你就无法分辨哪个数字才是真正的“众数”了。

------

### 4. 完整的计数逻辑（伪代码）

Python

```
if pre is None:      # 第一个节点
    count = 1
elif cur.val == pre.val: # 与前一个相同
    count += 1
else:                # 与前一个不同，新数字出现了
    count = 1

pre = cur # 更新前驱指针，为下一个节点做准备
```

------

### 5. 这种做法的巧妙之处：$O(1)$ 空间

通常找众数需要用 `HashMap`（字典）来存 `数字 -> 频率`，空间复杂度是 $O(n)$。

但利用了 **“有序性 + 相邻比较”**，我们只需要：

- 一个 `count`（当前频率）
- 一个 `maxCount`（最高频率记录）
- 一个 `pre` 指针
- 一个 `result` 数组

这样我们就把空间复杂度优化到了 **$O(1)$**（不计递归栈和结果集）。

------

### 总结

- **相邻比较**：是因为 BST 中序遍历是有序的，相同元素必堆在一起。
- **归 1**：是因为旧数字的任务结束了，新数字要重新从 1 开始报数。
